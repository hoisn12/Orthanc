# Plan Review

현재 대화에서 수립한 계획을 아래 체크리스트에 따라 리뷰한다.

## 컨텍스트 수집

- 현재 대화에서 사용자의 원래 요청사항을 확인
- 수립된 계획(변경 대상 파일, 구현 순서, 영향 범위)을 확인

## 체크리스트

- [ ] 사용자 요청사항이 계획에 빠짐없이 반영되었는가
- [ ] 변경 대상 파일 목록이 정확한가
- [ ] 기존 코드 패턴/유틸리티를 재사용하고 있는가 (불필요한 신규 코드 없는가)
- [ ] 구현 순서가 의존성을 고려하여 올바른가
- [ ] 엣지 케이스가 누락되지 않았는가
- [ ] 기존 아키텍처/컨벤션과 충돌하는 부분이 없는가
- [ ] 영향 범위가 과도하지 않은가 (최소 변경 원칙)

## 출력

리뷰 결과를 통과/미통과 항목으로 정리하여 보고한다.
미통과 항목이 있으면 계획 수정안을 제시하고, Plan 단계로 돌아가 수정한다.
