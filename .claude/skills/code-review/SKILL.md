# Code Review

Act 단계에서 변경된 코드를 리뷰한다.

## 컨텍스트 수집

- `git diff`로 변경된 코드 확인
- 변경된 파일의 전후 맥락을 읽어 변경 의도 파악

## 체크리스트

- [ ] 코드가 계획대로 정확히 구현되었는가
- [ ] 기존 코드 스타일/컨벤션을 준수하는가
- [ ] 불필요한 변경(포매팅, 주석, 리팩토링)이 포함되지 않았는가
- [ ] 변수/함수 네이밍이 명확하고 일관적인가
- [ ] 에러 핸들링이 적절한가 (과잉도 부족도 아닌가)
- [ ] 테스트가 필요한 변경인 경우 테스트가 포함되었는가
- [ ] 기존 테스트가 통과하는가 (`npm test` 실행)

## 출력

리뷰 결과를 통과/미통과 항목으로 정리하여 보고한다.
미통과 항목이 있으면 수정 사항을 구체적으로 제시하고, 수정 후 재리뷰한다.
